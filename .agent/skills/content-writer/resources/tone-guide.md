# トーン＆マナーガイド

ProductX AI DX記事のライティングスタイルを統一するためのガイドライン。

---

## 基本トーン

### ひとことで言うと
**「頼りになる先輩が、コーヒーを飲みながら解説してくれる」ようなトーン。**

### 3つのキーワード
1. **信頼感**: 専門家としての知見を感じさせるが、偉そうではない
2. **現場感**: 中小企業の現場を知っている人が書いている実感
3. **共感**: 読者の課題を「分かっている」姿勢

---

## 具体的なルール

### ✅ 使ってよい表現

| シーン | 表現例 |
|---|---|
| 問題提起 | 「〜していませんか？」「〜という経験はありませんか？」 |
| 共感 | 「多くの企業がこの課題に直面しています」「よくある悩みです」 |
| 提案 | 「〜という考え方があります」「〜するアプローチが有効です」 |
| 注意喚起 | 「ただし、注意すべき点があります」「ここが落とし穴です」 |
| CTA | 「まずは〜から始めてみてください」「お気軽にご相談ください」 |

### ❌ 避けるべき表現

| パターン | NG例 | OK例 |
|---|---|---|
| 過度な煽り | 「今すぐやらないと手遅れ！」「競合に負けます！」 | 「早期に検討を始めることで、より大きな効果が見込めます」 |
| 断定的すぎ | 「AIを導入すれば必ず成功します」 | 「適切に導入すれば、大きな成果が期待できます」 |
| 上から目線 | 「まだこんなことをやっているのですか」 | 「こうした課題は多くの企業に共通しています」 |
| カジュアルすぎ | 「めちゃくちゃ便利です！」「ヤバいです！」 | 「非常に効果的です」「大きなインパクトがあります」 |
| 専門用語の乱用 | 「LLMのfine-tuningでRAGを構築し…」 | 「AIに自社のデータを学習させることで…」 |

## 文体ルール

### 段落
- **1段落は3〜4行以内**（スマホでの可読性を重視）
- 長い説明は箇条書きに分解する

### 見出し
- **H2**: 記事の大きな構造（5〜7個が目安）
- **H3**: H2内の小トピック
- SEOキーワードを自然に含める（不自然なキーワード詰め込み禁止）

### 数字・データ
- 具体的な数字は**ソース付き**で提示
- Web検索で最新データを取得し、「〇〇調べ（2025年）」のように出典を明記
- 数字が取れない場合は「一般的に〜と言われています」で概算感を出す

### 太字・強調
- **太字**は1段落に1〜2箇所まで（多用すると効果が薄れる）
- 本当に伝えたいメッセージのみを太字にする

### 文末
- 「です・ます」調で統一
- 「〜です。」「〜ます。」「〜ください。」の連続を避ける（バリエーションを持たせる）

## ペルソナ別の書き分け

| ペルソナ | 響くワード | 響かないワード |
|---|---|---|
| **経営者** | コスト削減、売上向上、競合優位性、人材不足の解消 | 技術的詳細、API、データベース |
| **部門責任者** | 業務効率化、工数削減、チームの生産性、上への提案材料 | 経営戦略、全社DX |
| **DX推進担当** | 導入ステップ、失敗事例、PoC、社内説得の方法 | 基礎概念（既に知っている） |

> 記事のcategoryに応じて、主要ターゲットペルソナが変わる。
> - `sales` / `marketing` → 部門責任者が主、経営者が副
> - `backoffice` → 部門責任者が主
> - `dx-strategy` / `trend` → DX推進担当が主、経営者が副

## ProductXの言及ルール

- 記事本文内でProductXを**直接的に宣伝しない**
- 「ProductXを使えば〜」のような文言は**絶対に入れない**
- ProductXへの言及は**CTAセクションのみ**
- ただし「専門家に任せるべきポイント」セクションで、暗に専門家（=ProductX）の必要性を示す
