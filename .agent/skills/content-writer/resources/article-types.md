# 記事タイプ別ライティングガイド

4つのarticleTypeに応じた構成テンプレートと出し加減を定義。

---

## 1. awareness（課題啓発型）— 30本

### 目的
ターゲットに「うちも同じ課題がある」と気づかせる。

### 出し加減
- **What**: ★★★（惜しみなく出す）
- **How**: ★（軽く触れる程度）
- **CTA強度**: 低（認知フェーズ→関連記事への内部リンク中心）

### 推奨構成

```
## はじめに
（問いかけ or 衝撃的な事実で掴む。2〜3行で本記事の価値を伝える）

## [課題の具体描写]（例：「営業マンが売る以外に費やしている時間の正体」）
（読者が「あるある」と共感する具体的なシーンを3つ以上描写）
（可能であれば統計データや調査結果を引用）

## なぜこの課題が放置されるのか
（組織的・構造的な原因を分析。読者の視座を上げる）

## AIが変える可能性
（課題に対する「こんな世界が来ている」を概念レベルで提示）
（具体的ツールや手順は出さない）

## よくある誤解
（2〜3つの誤解を正す → 信頼感の醸成）

## まとめ：まず始める第一歩
（1つだけ、読者が今日できるアクションを提示）
（例：「まずは自社の業務を1週間記録してみてください」）

---

📩 関連記事: [関連記事タイトル1](リンク) / [関連記事タイトル2](リンク)

ProductXでは、AI DXに関する無料相談を承っています。
「うちの業務にAIは使えるのか？」という段階からお気軽にどうぞ。
[無料で相談する →](/contact)
```

### 注意点
- **統計データ**は必ずWeb検索で最新のものを取得
- 「あるある」のシーンはペルソナの業務に即した**現場感のある描写**にする
- CTAは控えめに。まず「関連記事」への内部リンクで回遊させる

## 2. comparison（事例・比較型）— 20本

### 目的
比較検討フェーズの読者に判断材料を提供し、「ProductXに相談しよう」に繋げる。

### 出し加減
- **What**: ★★★（選択肢と特徴を明確に）
- **How**: ★★（各選択肢のメリデメを概念レベルで）
- **CTA強度**: 中（記事中盤 + 末尾）

### 推奨構成

```
## はじめに
（比較のコンテキストを設定。読者が今まさに悩んでいるポイントに触れる）

## [選択肢A] の特徴とメリット・デメリット
（具体的な業務シーンとともに解説）

## [選択肢B] の特徴とメリット・デメリット
（同様に具体的に）

## [選択肢C / AI活用] の特徴と可能性
（AIを活用したアプローチ。ここが本記事のメインコンテンツ）

## 比較まとめ
（比較表を作成。コスト・時間・品質・スケーラビリティの4軸で）

| 項目 | 選択肢A | 選択肢B | AI活用 |
|---|---|---|---|
| コスト | ... | ... | ... |
| 時間 | ... | ... | ... |
| 品質 | ... | ... | ... |
| スケーラビリティ | ... | ... | ... |

## どの選択肢が最適か？
（「一概には言えない」→「自社の状況に合わせた判断が必要」）
（ここで「専門家に相談」への橋渡し）

## まとめ

---

📩 御社に最適な方法を一緒に探しませんか？
ProductXでは、AI DXに関する無料相談を承っています。
[無料で相談する →](/contact)
```

### 注意点
- **比較表は必須**。読者が一目で判断材料を得られるようにする
- AI活用の選択肢を「最強の正解」として出すのではなく、**公平に比較した上で自然にリードする**
- 費用感は「月額○万円〜」など幅を持たせた相場感で出す（Do領域に踏み込まない）

## 3. howto（ハウツー概念型）— 33本

### 目的
概念レベルの手順を提示し、「プロに頼んだ方が早い」を実感させる。

### 出し加減
- **What**: ★★（前提課題は簡潔に）
- **How**: ★★★（概念レベルの手順を詳しく）
- **CTA強度**: 高（記事中盤 + 末尾）

### 推奨構成

```
## はじめに
（このハウツーが解決する課題を1〜2文で明示）

## なぜ今、[テーマ]が注目されているのか
（市場トレンド・背景を簡潔に。統計データがあれば引用）

## [テーマ]の基本的な考え方
（概念レベルのフレームワークや手順を3〜5ステップで）
（各ステップは「何を」「なぜ」を中心に。「どうやって」は概念止め）

## 導入時に考慮すべきポイント
（落とし穴・注意点を3つ程度）
（読者が「確かに自分では難しい」と思うレベルの情報）

## 専門家に任せるべきポイント
（ここが最重要セクション）
（「概念の理解は自社でもできるが、実装は専門家と共に」のメッセージ）
（具体的に何がプロの領域かを明示）

## まとめ

---

📩 導入を検討中の方へ
ProductXでは、AI DXの無料個別相談を承っています。
御社の業務フローに合わせた具体的なご提案をいたします。
[無料で相談する →](/contact)
```

### 注意点
- **「専門家に任せるべきポイント」セクションは必須**。これがコンバージョンの要
- 手順は「概念レベル」に留める。具体的なツール名やコードは出さない
- ステップ内にビジュアル（フロー図など）を入れると理解度が上がる

## 4. roi（ROI・判断型）— 17本

### 目的
意思決定者の「やる/やらない」の判断を助け、「やるなら相談」に繋げる。

### 出し加減
- **What**: ★★（前提は簡潔に）
- **How**: ★★（費用感・期間は幅を持たせて提示）
- **CTA強度**: 最高（記事冒頭 + 中盤 + 末尾）

### 推奨構成

```
## はじめに
（意思決定のプレッシャーに共感する書き出し）

> 📩 今すぐROI試算をご希望の方は[無料相談](/contact)からお問い合わせください。

## 費用対効果を考える3つの軸
（※ トピック固有の軸に置き換える）
1. 直接的なコスト削減（数字の例を記事固有のものに）
2. 時間の再配分による価値創出（具体的な業務シーンで示す）
3. リスク低減の価値（属人化・ミス・対応遅延）

## [トピック固有]の費用相場
（「月額○万円〜○万円が一般的」 など幅のある相場感）
（初期費用・月額費用・立ち上げ期間を分けて提示）

## よくある試算ミス
（3つ程度。読者の「自分で計算した」結果を疑わせる→専門家の必要性）

## 正しいROI評価のために
（現状の業務コスト把握の重要性 → 業務棚卸しの推奨）

## 判断チェックリスト
（3〜5項目の自己診断。「はい/いいえ」で回答 → 結果に応じたアドバイス）

## まとめ

---

📩 御社の場合のROIを無料で試算します
ProductXでは、AI DXに関する無料のROI試算サービスを提供しています。
[無料で相談する →](/contact)
```

### 注意点
- **数字は記事ごとにトピック固有のものを使う** → テンプレート使い回し厳禁
- ROI型はCTAが最も強い→記事冒頭にも「今すぐ知りたい方」向けのCTAを入れる
- 費用相場は**幅を持たせる**（Do領域に踏み込まない）
- 「判断チェックリスト」は読者のエンゲージメントを高める効果がある
