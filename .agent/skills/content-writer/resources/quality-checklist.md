# 品質チェックリスト

リライトした記事が満たすべき最低品質基準。セルフレビュー完了後、最終チェックとして使用する。

---

## Frontmatter

- [ ] `title`: 30〜40文字、ターゲットSEOキーワードを含む
- [ ] `description`: 120文字以内、課題→解決の要約
- [ ] `date`: リライト実施日に更新
- [ ] `category`: 5つのいずれか（sales / marketing / backoffice / dx-strategy / trend）
- [ ] `tags`: 3つ程度、検索されやすいキーワード
- [ ] `articleType`: 4つのいずれか（awareness / comparison / howto / roi）
- [ ] `relatedServices`: 対応するサービスカタログ番号を設定（該当なければ空配列）

## 構成

- [ ] 記事タイプ別テンプレートの推奨構成に沿っている
- [ ] H2見出しが5〜7個ある
- [ ] H2/H3にロングテールKWが自然に含まれている
- [ ] 「はじめに」セクションが2〜3行で本記事の価値を伝えている
- [ ] 「まとめ」セクションがある

## 本文品質

- [ ] テンプレート文言の使い回しがない（他の記事と本文が重複していない）
- [ ] トピック固有の具体的な内容がある（業務シーン、数値、事例）
- [ ] 黄金比率（What80%/How20%/Do0%）が守られている
- [ ] 具体的なツール名・プロンプト・設定手順・コードが含まれていない（Do 0%）
- [ ] 1段落が3〜4行以内
- [ ] 専門用語には説明がある
- [ ] 太字は1段落に1〜2箇所以内

## SEO

- [ ] タイトルにメインKWが含まれている
- [ ] H2の少なくとも1つにKWバリエーションが含まれている
- [ ] meta description（frontmatter.description）が魅力的で検索結果でクリックしたくなる
- [ ] 本文中にKWが自然に3〜5回出現する（キーワード詰め込みではない）

## コンバージョン

- [ ] 記事タイプに応じたCTA強度になっている
- [ ] CTAに問い合わせフォーム（`/contact`）への導線がある
- [ ] howto型の場合：「専門家に任せるべきポイント」セクションがある
- [ ] roi型の場合：記事冒頭にもCTAがある

## 内部リンク

- [ ] 関連記事へのリンクが3〜5本ある
- [ ] リンク先のslugが実在する
- [ ] カテゴリ内の関連記事へのリンクが中心
- [ ] ピラーページ（what-is-ai-dx / genai-business-guide-2026）へのリンクが1本以上ある

## ビジュアル

- [ ] 記事内に最低1つのビジュアル要素がある（図、比較表、画像のいずれか）
- [ ] 画像パスが正しい（`/articles/ai-dx/images/xxx.webp`）
- [ ] 画像に代替テキスト（alt属性）がある
- [ ] 比較表がある場合、Markdown table記法を使っている

## トーン＆マナー

- [ ] 「です・ます」調で統一されている
- [ ] 過度な煽り・断定・カジュアルな表現がない
- [ ] ProductXの直接的な宣伝が本文中にない（CTAセクションのみ）
