## はじめに

![比較分析](/articles/ai-dx/images/comparison-hero.webp)

「RPAを導入したが、半年で止まってしまった」——こんな失敗談は珍しくありません。ある調査によると、RPA導入企業の**約40%が1年以内に運用を停止**しています。

この記事では、RPAが止まる3つの構造的な理由を深堀りし、**AIワークフローがなぜそれを解決できるのか**を比較解説します。

## RPAが止まる3つの構造的理由

### 理由1: 画面変更で壊れる

RPAは画面上のボタン位置やHTML要素名を**座標レベルで記録**して動作します。UIが変わると即停止。

| 原因 | 頻度 | 修復コスト |
|---|---|---|
| Windowsアップデート | 年数回 | 数時間〜数日 |
| WebサービスのUI更新 | 年数回 | 数時間〜数日 |
| ブラウザのバージョンアップ | 年2〜4回 | 数時間 |
| モニター解像度の変更 | 稀 | 1〜2時間 |

年間で**10〜30回のメンテナンス**が発生するケースも。

### 理由2: 例外処理に弱い

RPAは事前に定義されたルール通りにしか動けません。

| 例外パターン | RPAの反応 |
|---|---|
| いつもと違うフォーマットの請求書 | 停止 |
| 想定外のエラーメッセージ | 停止 |
| ログイン時のCAPTCHA表示 | 停止 |
| データ項目の追加 | 停止 |
| 承認者が不在（通常と異なるワークフロー） | 停止 |

### 理由3: スケールしない

| 自動化業務数 | メンテナンス工数/月 | 年間コスト |
|---|---|---|
| 3業務 | 5時間 | 21万円 |
| 5業務 | 12時間 | 50万円 |
| 10業務 | 30時間 | 126万円 |
| 20業務 | 80時間以上 | 336万円以上 |

10業務以上でフルタイム1人分のメンテナンスリソースが必要に。

## AIワークフローはなぜ止まらないのか

### 根本的な違い

| アプローチ | RPA | AIワークフロー |
|---|---|---|
| 動作の基盤 | 画面のUI | **API、データ構造** |
| 処理の方法 | 手順の記録・再生 | **目的の理解と判断** |
| 変化への対応 | UI変更で停止 | **UIに依存しない** |
| 例外処理 | 停止する | **AIが判断して処理** |
| 学習能力 | なし | **あり** |

### 14項目の詳細比較

| 項目 | RPA | AIワークフロー |
|---|---|---|
| 画面変更への耐性 | ✕ | ◎ |
| 例外処理 | ✕ | ○ |
| 学習機能 | なし | あり |
| 定型業務 | ◎ | ◎ |
| 非定型業務 | ✕ | ○ |
| メンテナンスコスト | 高（年々増加） | 低（年々減少） |
| スケール性 | 困難 | 容易 |
| 初期導入コスト | 中程度 | 中〜やや高め |
| TCO（3年間） | 高い | 低い |
| 精度の変化 | 不変 | 年々向上 |
| 導入期間 | 1〜3ヶ月/業務 | 1〜2ヶ月/業務 |
| 必要スキル | RPAスキル | API連携知識 |
| レガシー対応 | ◎ | △ |
| 判断を含む処理 | ✕ | ○ |

## どちらを選ぶべきか

### RPAが適しているケース

- 対象業務が**完全に定型的**（例外が月1回以下）
- 対象システムの**UIが長期間変わらない**
- APIがないレガシーシステム
- 自動化業務が**1〜3個に限定**

### AIワークフローが適しているケース

- 例外パターンが**月5回以上**発生
- 対象システムの**UIが頻繁に変更**
- **5個以上の業務**を自動化したい
- 判断を含む処理（分類、抽出、要約等）がある
- **3年以上の長期運用**を予定

## RPAからAIワークフローへの移行ステップ

### ステップ1: RPA棚卸し

| 分類 | 対応 |
|---|---|
| 安定稼働中 | そのまま維持 |
| 頻繁に止まる（月1回以上） | 優先的にAI化 |
| 完全停止（放置中） | 即座にAI化を検討 |

### ステップ2: API連携の可能性を確認

AIワークフローに移行するには対象システムのAPI提供が必要。APIがないシステムは引き続きRPAで対応。

### ステップ3: 頻繁に止まる業務から移行

RPAが頻繁に停止する業務は、AIワークフローへの移行優先度が最も高い。

### ステップ4: 段階的に拡大

1つの成功体験が次の移行を加速させます。

## まとめ

RPAは「定型×安定」な業務には有効ですが、現実の業務は**「例外だらけ×変化が多い」**。RPAが止まる3つの理由は構造的な問題であり、努力では解決できません。

AIワークフローは、これらを**技術的に解消**した次世代の業務自動化です。
